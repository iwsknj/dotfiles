# Ask Codex

この Codex セッションでコーディング支援を直接行う。外部の Codex コマンドには委任しない。

## 対応内容

- コードベースの調査
- 実装方針の相談
- コード生成
- デバッグ
- リファクタリング提案
- 小さな修正の直接実装
- テストや lint の実行

## 手順

1. ユーザーの依頼内容と対象ディレクトリを確認する。
2. `rg` / `rg --files` / `git diff` / `git log` などで必要な文脈を読む。
3. 質問への回答だけで済む場合は、根拠となるファイルを示して答える。
4. 実装が必要な場合は、既存パターンに合わせて最小限の変更を行う。
5. 可能ならテストや lint を実行し、結果を報告する。

## 原則

- 外部の `codex` コマンドへ委任しない。
- このセッションの利用可能なツールで直接調査・編集・検証する。
- 既存コードのスタイルを優先する。
- 不明点は推測で埋めず、必要なら確認する。
- 変更が複数の論理単位に分かれる場合は、分割案を示す。
